## Usage

<code>[CyclicEdge]()[*v1*, *v2*]</code> represents the hyperedge `{`*v1*`,` *v2*`}` with cyclic edge symmetry, given in the edge list passed to [Hypergraph]().

<code>[CyclicEdge]()[*v1*, *v2*, *tag*]</code> represents the same hyperedge, tagged with *tag*.

<code>[CyclicEdge]()[{*v1*, …, *vk*}]</code> represents the *k*-ary hyperedge `{`*v1*`, …,` *vk*`}` with cyclic edge symmetry.

## Details & Options

- [CyclicEdge]() is one of three heads [Hypergraph]() recognizes as edge-symmetry markers in an edge specification: [DirectedEdge]() gives `"Ordered"` symmetry, [UndirectedEdge]() gives `"Unordered"` symmetry (the full symmetric group on the edge's vertices), and [CyclicEdge]() gives `"Cyclic"` symmetry (the cyclic group generated by rotating the vertex list).
- The cyclic group of a *k*-ary edge has *k* elements, the identity and its *k - 1* nontrivial rotations; for *k* = 2 this coincides with `"Unordered"` symmetry.
- Outside an edge specification, [CyclicEdge]() carries no evaluation rules of its own and remains an unevaluated symbolic expression, the same way [DirectedEdge]() and [UndirectedEdge]() do.
- [EdgeSymmetry]() reports `"Cyclic"` for a hyperedge specified with [CyclicEdge](); [HyperedgeList]() reconstructs it as a [Hyperedge]() tagged `"Cyclic"`.

## Basic Examples

Build a hypergraph with a cyclic ternary hyperedge:

```wl
hg = Hypergraph[{CyclicEdge[{1, 2, 3}]}]
```

<!-- => Hypergraph object, rendered as a hypergraph plot with 3 vertices -->

Its edge symmetry is reported as `"Cyclic"`:

```wl
EdgeSymmetry[hg]
```

<!-- => {"Cyclic"} -->

The full edge symmetry expands into the 3 elements of the cyclic group, the identity and the two nontrivial rotations:

```wl
hg["FullEdgeSymmetry"]
```

<!-- => {{Cycles[{}], Cycles[{{1, 2, 3}}], Cycles[{{1, 3, 2}}]}} -->

## Scope

For an arity-2 edge, `"Cyclic"` and `"Unordered"` symmetry give the same full edge symmetry:

```wl
EdgeSymmetry[Hypergraph[{CyclicEdge[1, 2]}]] === EdgeSymmetry[Hypergraph[{UndirectedEdge[1, 2]}]]
```

<!-- => True -->

---

A tagged cyclic edge is given with 3 arguments, the two vertices followed by the tag:

```wl
EdgeSymmetry[Hypergraph[{CyclicEdge[1, 2, "x"]}]]
```

<!-- => {{Cycles[{}], Cycles[{{1, 2}}]}} -->

## Properties and Relations

A cyclic ternary hyperedge contributes only the 3 edges of a directed triangle to [HypergraphToGraph](), one for each rotation:

```wl
Union[EdgeList[HypergraphToGraph[Hypergraph[{CyclicEdge[{1, 2, 3}]}]]]]
```

<!-- => {DirectedEdge[1, 2], DirectedEdge[2, 3], DirectedEdge[3, 1]} -->

---

The same ternary hyperedge without [CyclicEdge]() defaults to `"Unordered"` symmetry, whose full symmetric group contributes every directed pair among the 3 vertices:

```wl
Union[EdgeList[HypergraphToGraph[Hypergraph[{{1, 2, 3}}]]]]
```

<!-- => {DirectedEdge[1, 2], DirectedEdge[1, 3], DirectedEdge[2, 1], DirectedEdge[2, 3], DirectedEdge[3, 1], DirectedEdge[3, 2]} -->
